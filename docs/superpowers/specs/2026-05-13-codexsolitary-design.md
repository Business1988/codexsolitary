# CodexSolitary Design Spec

## Goal

Create a mobile software and IAA monetization studio website for CodexSolitary that feels like a systems console or public operations terminal, not a polished agency brochure.

## Brand and Contact

- Domain: codexsolitary.com
- VIP: vip@codexsolitary.com — Key Account Office
- Business: business@codexsolitary.com — Business Development
- Support: support@codexsolitary.com — Technical Assistance

## Visual Language

- Tone: technical, structural, precise, terminal-like
- Palette: graphite black, cold gray, ice blue
- Layout: split panels, system strips, command-stack sections, dense but readable
- Avoid: warm editorial styling, magazine rhythm, product showcase softness

## Structure

- `index.html` — systems overview
- `systems.html` — operating architecture
- `release-stack.html` — production/release layers
- `revenue-ops.html` — IAA monetization operations
- `contact.html`
- `privacy.html`
- `support.html`

## Differentiators

- Feels like an external-facing command center
- More modular, less narrative
- Heavy use of labels, codes, system grouping
